# PRD: pos-voice-concierge

## 概要

open-posに音声インターフェースを追加する拡張プラグイン。店員が「○○を3つ」と話すだけでバーコードスキャン不要で商品を登録でき、「今日の売上は？」と聞けば即座に集計を読み上げる。日本語の商品名の表記ゆれをファジーマッチングで吸収し、小規模店舗のワンオペ時に両手がふさがっていても会計処理を進められる。

## 問題定義

### ターゲットユーザー
- 小規模小売店（個人商店、地域のコンビニ、専門店）のワンオペ店員
- 両手がふさがる作業（陳列、梱包、調理）中に会計処理が必要な場面

### 解決する課題
1. **ワンオペ時の会計停滞**: 両手が商品やダンボールで塞がっているとき、バーコードスキャンのために一度手を止める必要がある
2. **売上確認の非効率**: 売上や在庫の確認にPOS画面への遷移操作が必要
3. **バーコードなし商品**: 量り売り、手作り商品など、バーコードがない商品の登録が煩雑

### ユニークポイント
日本の小規模小売店特有の「ワンオペ+両手ふさがり」問題を解決し、声だけで会計から売上確認まで完結できる点。

## 競合分析

| 競合 | 特徴 | 弱点 |
|------|------|------|
| Alexa for Business | 音声でビジネスタスクを実行 | POS連携なし、日本語商品名の表記ゆれ非対応、月額課金 |
| Google Assistant for POS | Google Cloud音声認識基盤 | POS専用ではない、カスタム語彙の学習機能なし |
| Siri Shortcuts + Square | iOSショートカットでSquare操作 | Apple端末限定、日本語ファジーマッチング非対応、柔軟性低い |
| Voysis (Walmart) | 大規模小売向け音声検索 | エンタープライズ向け、小規模店舗には過剰、高コスト |

### 差別化ポイント
- **open-pos統合**: 既存のopen-posプラグインAPIに直接統合。追加ハードウェア不要
- **日本語特化ファジーマッチング**: 商品名の表記ゆれ（「コカコーラ」「コカ・コーラ」「Coca-Cola」）を吸収
- **自己学習辞書**: 店舗固有の商品名・略称を学習し精度向上
- **ゼロコスト音声認識**: Whisperによるオンプレミス処理で従量課金なし

## MVP機能要件

### F-1: 音声商品登録
**概要**: マイク入力からの音声認識と商品名ファジーマッチングによるPOS商品登録

**受け入れ条件**:
- [ ] マイクボタン押下で音声入力を開始し、「○○を3つ」形式の発話を認識する
- [ ] 認識された商品名をファジーマッチングで既存商品マスタと照合する（閾値: 類似度80%以上）
- [ ] マッチ候補が複数ある場合、上位3件を表示して選択させる
- [ ] 確定後、指定数量でopen-posのカートに商品を追加する
- [ ] 認識失敗時は音声フィードバック（「聞き取れませんでした」）を返す
- [ ] 1発話の処理レイテンシが3秒以内

### F-2: 自然言語クエリ（売上・在庫照会）
**概要**: 「今日の売上」「○○の在庫」等の自然言語クエリに対する音声レスポンス

**受け入れ条件**:
- [ ] 「今日の売上は？」で当日の売上合計金額を音声で読み上げる
- [ ] 「○○の在庫は？」で指定商品の在庫数を音声で読み上げる
- [ ] 「今月の売上トップ5」で売上上位商品を読み上げる
- [ ] クエリ意図の認識精度が90%以上（テストセット基準）
- [ ] レスポンス生成が2秒以内

### F-3: 確認/修正UIと表記ゆれ辞書学習
**概要**: 認識結果の確認/修正UIと商品名の表記ゆれ辞書の学習機能

**受け入れ条件**:
- [ ] 音声認識結果と商品マッチング結果をリアルタイムで画面表示する
- [ ] 誤認識時にタッチ操作で正しい商品を手動選択できる
- [ ] 手動修正された組み合わせを表記ゆれ辞書に自動登録する
- [ ] 辞書データはPostgreSQLに永続化し、次回以降のマッチング精度を向上させる
- [ ] 辞書のエクスポート/インポート機能（JSON形式）

## 技術リスク

| リスク | 影響度 | 軽減策 |
|--------|--------|--------|
| 日本語商品名の音声認識精度 | 高 | Whisperの日本語モデル使用 + 商品名語彙によるファインチューニング検討 |
| 店舗内騒音環境での認識率低下 | 高 | ノイズキャンセリング前処理 + 指向性マイク推奨 + 認識閾値の動的調整 |
| 音声認識〜商品登録のレイテンシ | 中 | gRPCストリーミングによるリアルタイム処理 + Whisper tiny/baseモデル選択 |
| 表記ゆれ辞書の冷起動問題 | 中 | 初期辞書としてJANコードマスタのカタカナ読みを事前登録 |
| ブラウザのマイクAPI互換性 | 低 | Web Audio API + MediaRecorder のPolyfill、Chrome/Edge優先対応 |

## 技術スタック

| 層 | 技術 | 用途 |
|----|------|------|
| backend | Kotlin/Quarkus | open-posプラグインAPI + gRPCサービス |
| voice-engine | Python (Whisper) | 音声認識 + ファジーマッチング |
| frontend | TypeScript/React | 音声UI + 商品確認画面 |
| 通信 | gRPC | 音声ストリーミング |
| インフラ | GCP Cloud Run | サーバーレスデプロイ |
| DB | PostgreSQL | 商品マスタ + 表記ゆれ辞書 |

## 非機能要件

| 項目 | 要件 |
|------|------|
| レイテンシ | 音声入力〜商品登録: p95 < 3s |
| 可用性 | 音声エンジン障害時はフォールバック（通常POS操作）で100%業務継続 |
| セキュリティ | PCI-DSS準拠（カード情報の音声入力は禁止）、音声データの非永続化 |
| プライバシー | 音声データはストリーミング処理のみ、サーバー側に保存しない |
| ブラウザ | Chrome 90+, Edge 90+ |
